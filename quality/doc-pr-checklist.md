# Documentation PR Checklist

## Pre-Merge Checklist

- [ ] Scored against the quality rubric (≥ 14/20)
- [ ] No broken links (CI link-check pass)
- [ ] Markdown lint pass (no errors)
- [ ] Spell-check pass
- [ ] Reviewed by at least one other team member
- [ ] Audience metadata tag applied
- [ ] Diátaxis classification correct (Tutorial / How-to / Reference / Explanation)
- [ ] Code samples are tested and runnable
- [ ] Sensitive information (PII, credentials) checked — none present

## Post-Merge

- [ ] Published to documentation site
- [ ] Added to navigation if applicable
- [ ] Related team members notified

## Common Issues (Aghajani Taxonomy Reference)

| Issue | Category | Check |
|-------|----------|-------|
| Missing example | Completeness | Does the doc include a working example? |
| Outdated screenshot | Currency | Is every visual up to date? |
| Ambiguous instruction | Clarity | Can a new team member follow this? |
